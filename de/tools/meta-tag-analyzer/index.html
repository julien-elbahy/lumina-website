So funktioniert dieses Tool

Gib eine beliebige öffentliche URL ein und das Tool ruft die Seite über unsere eigenen Server ab — keine Browser-Erweiterung nötig. Es parst den vollständigen HTML-Code und extrahiert alle SEO-relevanten Meta-Tags: Title, Description, Canonical, Robots, Open Graph, Twitter Cards, Schema.org JSON-LD, hreflang, Viewport, Charset und mehr.

Die Ergebnisse sind in 6 Tabs organisiert: On-Page SEO (Title, Description, H1, Canonical, Robots), Heading-Counts, Open Graph Tags, Twitter Card Tags, Schema.org Detection und technisches Setup (hreflang, Viewport, Charset, Lang, Favicon). Ein gewichteter SEO-Score (0–100) gibt dir sofort einen Überblick, wie gut die Seite optimiert ist — mit umsetzbaren Tipps für jedes gefundene Problem.

Bei JavaScript-lastigen Seiten wechselt das Tool automatisch zu einem Headless-Browser-Renderer, um genaue Ergebnisse zu liefern — genau so, wie eine Suchmaschine die Seite sehen würde.

Dieses Tool ist ein reines Diagnose-Audit. Es zeigt dir, was auf der Seite ist, was fehlt und was kaputt ist. Wenn du deinen Title und deine Description umschreiben willst — mit Live Google-Snippet-Vorschau — nutze den SERP Preview Editor. Für visuelle Social-Card-Vorschauen den OG Preview. Jedes Spezial-Tool geht tiefer, als es ein generisches All-in-One-Audit je könnte.

FAQ

Welche Meta-Tags prüft dieses Tool?+
Title, Description, Canonical, Robots, H1, alle Open Graph Tags (og:title, og:description, og:image, og:url, og:type, og:site_name, og:locale), Twitter Card Tags, Schema.org JSON-LD, hreflang, Viewport, Charset, Lang-Attribut und Favicon.
Ist dieses Tool kostenlos?+
Komplett kostenlos, ohne Anmeldung. Die Analyse läuft über unsere eigenen Server — keine Drittanbieter-Proxies. Du kannst unbegrenzt viele URLs analysieren.
Wie wird der SEO-Score berechnet?+
Der Score startet bei 100 und nutzt tier-basierte Abzüge. Kritische Fehler (fehlender Title/Description, noindex, HTTP-Fehler) capen den Score automatisch auf 15–40, unabhängig von den anderen Checks. Wichtige Probleme (fehlender H1, schlechtes Canonical, kaputte OG-Tags, Pixel-Width-Overflow) ziehen jeweils 4–10 Punkte ab. Nice-to-have (Viewport, Lang, Charset, Schema) ziehen 2–3 Punkte ab. Duplikate und Meta-Tag-Konflikte aus dem Issues-Banner kosten je 2–6 weitere Punkte. Das Scoring ist auch seitentyp-bewusst: Homepages dürfen kurze Brand-Titles haben, Artikel werden auf artikel-spezifische Signale geprüft. Ein Score von 80+ bedeutet das Meta-Setup ist solide; 50–79 verbesserbar; unter 50 hat kritische Probleme.
Funktioniert dieses Tool mit JavaScript-lastigen Websites?+
Ja. Wenn der erste Abruf unvollständige Daten liefert (häufig bei React-, Angular- oder Vue-Apps), wechselt das Tool automatisch zu einem Headless-Browser, der JavaScript rendert — so siehst du die Seite genau wie Suchmaschinen.
Kann ich meine Meta-Tags hier mit KI umschreiben lassen?+
Dieses Tool ist ein reines Diagnose-Tool — es zeigt dir was auf der Seite ist, was fehlt und was kaputt ist. Zum Umschreiben nutzt du den SERP Preview: dort gibt's einen Live-Google-Snippet-Editor mit Desktop- & Mobile-Vorschau, Keyword-Highlighting und KI-basierten Title- & Description-Vorschlägen. Nach jeder Analyse zeigen wir einen „Im SERP Preview öffnen“-Button, der die URL direkt übergibt.
Warum ist die Pixelbreite bei Titeln und Beschreibungen wichtig?+
Google kürzt Suchergebnisse basierend auf Pixelbreite, nicht Zeichenanzahl. Ein Titel mit breiten Zeichen wie „W" oder „M" wird früher abgeschnitten als einer mit schmalen Zeichen. Dieses Tool misst die tatsächlich gerenderten Pixel für genauere Vorhersagen als reines Zeichenzählen.
Kann ich Seiten hinter einem Login analysieren?+
Nein, dieses Tool kann nur öffentlich zugängliche Seiten analysieren. Für Seiten hinter einer Authentifizierung nutze die Lumina Chrome-Erweiterung, die direkt in deinem Browser läuft und jede Seite analysieren kann, auf die du Zugriff hast.
