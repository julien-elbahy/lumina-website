hreflang Checker

Validiere hreflang-Tags auf jeder Seite. Prüfe Sprach-/Regions-Annotationen, verifiziere Rück-Links und erkenne häufige Implementierungsfehler.

So funktioniert es

Gib eine URL ein und das Tool holt den HTML-Code der Seite und extrahiert alle <link rel="alternate" hreflang="...">-Tags. Es validiert Sprachcodes, prüft auf selbstreferenzierende Tags, x-default und ermöglicht die Verifizierung, ob alternative Seiten korrekt zurückverlinken (Return-Link-Validierung).

Häufig erkannte Probleme

• Fehlender selbstreferenzierender Tag (die Seite sollte auf sich selbst verweisen)
• Fehlendes x-default (sollte eine Fallback-Sprache angeben)
• Nicht-standardisierte Sprachcodes (z.B. „uk" statt „en-GB")
• Fehlende Return-Links (alternative Seiten verlinken nicht zurück)
• HTTP/HTTPS-Abweichungen zwischen hreflang-URLs

Was ist hreflang?+
hreflang ist ein HTML-Attribut, das Suchmaschinen mitteilt, welche Sprache und Region eine Seite anspricht. Es hilft, Duplicate-Content-Probleme bei mehrsprachigen Seiten zu vermeiden und stellt sicher, dass Nutzer die richtige Sprachversion in den Suchergebnissen sehen.
Brauche ich einen selbstreferenzierenden hreflang-Tag?+
Ja. Google verlangt, dass jede Seite mit hreflang einen selbstreferenzierenden Tag enthält, der auf sich selbst zeigt. Ohne diesen ignoriert Google möglicherweise alle hreflang-Annotationen auf der Seite.
Was ist x-default?+
x-default gibt die Fallback-URL für Nutzer an, deren Sprache/Region keinem hreflang-Tag entspricht. Es wird für alle mehrsprachigen Implementierungen dringend empfohlen.
Können hreflang-Fehler Rankings beeinträchtigen?+
Ja. Ungültige hreflang-Tags, fehlende Rückverweise oder falsche Sprachcodes können dazu führen, dass Google alle hreflang-Annotationen einer Seite ignoriert. Das kann dazu führen, dass die falsche Sprachversion in den Suchergebnissen erscheint oder Duplicate-Content-Probleme entstehen.
Wie viele hreflang-Tags brauche ich?+
Du brauchst einen hreflang-Tag für jede Sprach- oder Regionalversion einer Seite, plus einen selbstreferenzierenden Tag für die aktuelle Seite. Ein x-default-Tag für die Fallback-Version wird ebenfalls dringend empfohlen.
