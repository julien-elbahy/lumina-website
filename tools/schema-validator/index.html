How this validator works

This tool parses your JSON-LD and checks it against Google's documentation for structured data. Unlike syntax-only validators, it also checks whether required fields are present for each schema type, whether dates use ISO 8601 format, whether URLs are valid, and whether nested objects (like author or publisher) have their own required fields.

For even deeper validation — checking your schema against actual page content (e.g., does the price in your Product schema match the price on the page?) — use the Lumina Chrome Extension which cross-references schema data with live page content.

Frequently asked questions

What schema types does this validator support?+
It validates the most common types used for Google Rich Results: Article, Product, FAQPage, LocalBusiness, Organization, Person, BreadcrumbList, Event, Recipe, HowTo, VideoObject, and WebSite. Each type has its own set of required and recommended fields.
How is this different from Google's Rich Results Test?+
Google's Rich Results Test checks if your schema qualifies for specific rich result types. Our tool focuses on field-level validation — checking required fields, data formats, and nested objects. Both tools complement each other. We provide a direct link to Google's tool for each validation.
Can I validate schema from a live URL?+
This web tool works with pasted JSON-LD. For live URL validation, install the free Lumina Chrome Extension — it automatically extracts and validates all JSON-LD schemas on any page you visit.
